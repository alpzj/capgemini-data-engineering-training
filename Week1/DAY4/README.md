# SQL Data Cleaning, Transformation & Pattern Extraction Project

## 📌 Overview
This project is part of my ongoing **Capgemini Databricks Training**, focused on strengthening practical SQL skills used in real-world data engineering and analytics workflows.  

The primary objective of this work is to demonstrate how to handle **missing values, perform conditional transformations, and extract meaningful patterns from structured text data** using SQL.

---

## 🎯 Objectives
- Understand and handle missing (NULL) values in datasets
- Apply data transformation techniques for cleaning and standardization
- Use conditional logic for deriving meaningful business insights
- Extract structured information from unstructured or semi-structured text
- Practice advanced SQL pattern matching techniques

---

## 🗂️ Datasets Used
The project is built using three main relational datasets:

- **Employee Data**: Contains employee details such as salary, bonus, and manager relationships with missing values.
- **Orders Data**: Includes customer orders with attributes like amount, discount, and coupon codes, with partial missing data.
- **Products Data**: Contains product details including price, category, and stock with incomplete records.
- **Regex Practice Dataset**: A structured dataset designed specifically for practicing text parsing, pattern recognition, and extraction techniques.

---

## 🛠️ Key SQL Concepts Applied

### 1. Handling Missing Values
- Identifying NULL values using conditional filtering
- Replacing NULLs with meaningful defaults using `COALESCE`
- Conditional logic using `NULLIF`

### 2. Data Standardization
- Converting inconsistent numeric fields into standardized formats
- Ensuring clean and usable numeric outputs for analysis

### 3. Business Logic Implementation
- Calculating total earnings using salary and bonus combinations
- Computing payable amounts after applying discounts
- Handling edge cases such as missing or zero values

### 4. Pattern Extraction Using Regular Expressions
- Extracting numeric sequences from mixed text fields
- Capturing prefixes, suffixes, and embedded values
- Extracting email usernames and domain names
- Isolating country codes from phone numbers
- Identifying structured codes within compound identifiers

### 5. Text Processing Techniques
- Extracting first and last characters from strings
- Splitting structured identifiers into meaningful components
- Parsing alphanumeric patterns for analytics use cases
